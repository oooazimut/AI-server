# Глава: разбор обратной связи

## Назначение

Используй эту главу, когда пользователь сообщает о плохом ответе системы или указывает конкретную ошибку.

## Что определить

1. Какой результат ожидал пользователь.
2. Что система ответила фактически (поле `response` в событии).
3. Почему пользователь считает ответ неправильным.
4. Это ошибка понимания, маршрутизации, инструмента, данных или финальной формулировки.

## Как найти событие по жалобе

1. Используй `diagnost_search_events` с ключевыми словами из жалобы.
2. Если пользователь знает приблизительный запрос — ищи по этим словам.
3. Если нашёл несколько событий — уточни у пользователя.
4. Если не нашёл — проверь `diagnost_list_incidents` (статус open).

## Правила анализа

- Жалоба сама по себе не объясняет причину сбоя — нужно связанное событие.
- Комментарий человека важнее автоматического вывода.
- Если событие найдено, проверь `status`, `confidence`, `handoff_to`, `actions`.
- Если данных нет, явно укажи, каких данных не хватает.

## Выход

Сформулируй коротко:

- суть жалобы;
- найденное событие (event_id);
- первичную категорию ошибки (маршрутизация / tool / LLM / данные / финальный ответ).
