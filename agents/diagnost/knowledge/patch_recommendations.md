# Глава: рекомендации исправлений

## Назначение

Используй эту главу, когда нужно предложить, что именно исправить после разбора ошибки.

## Возможные типы исправлений

| Тип | Файл / место |
|---|---|
| Инструкции агента | `agents/<id>/instructions.md` |
| Skill агента | `agents/<id>/skills/<name>.md` |
| Знание агента | `agents/<id>/knowledge/<name>.md` |
| AgentTool | `backend/ai_server/agents/<id>/tools/<name>.py` |
| Код обработки результата | `backend/ai_server/agents/<id>/specialist.py` |
| Regression test | `tests/test_<module>.py` |

## Как выбирать исправление

- Если Переговорщик неправильно маршрутизировал → проверь `instructions.md` Переговорщика или skill маршрутизации.
- Если специалист вызван, но действовал неправильно → проверь его `instructions.md` и skills.
- Если tool вызван с неверными аргументами → проверь skill специалиста и код инструмента.
- Если tool вернул корректные данные, но ответ плохой → проверь compose/final answer в LLM-сервисе.
- Если ошибка повторяется систематически → предложи общий патч и regression test.

## Ограничения

ИИ-Диагност не применяет патч сам. Он только предлагает исправление с обоснованием.

Для применения требуется отдельное решение человека.
