# Глава: групповой анализ ошибок

## Назначение

Используй эту главу, когда нужно разобрать несколько похожих ошибок вместе или построить сводку за период.

## Как использовать diagnost_error_report

`diagnost_error_report(since_hours=24)` возвращает:

| Поле | Описание |
|---|---|
| `total_incidents` | общее количество инцидентов за период |
| `open_incidents` | открытые (не разрешённые) |
| `by_reason` | группировка: failed / low_confidence / manual |
| `recent_incidents` | последние 10 с event_id, request, confidence |

## Как группировать

1. По `reason` — `failed` vs `low_confidence` vs `manual`.
2. По ключевым словам в `request` — однотипные запросы.
3. По временному интервалу — пики ошибок (увеличь `since_hours`).

## Что искать

- один и тот же паттерн запроса → неправильный routing к специалисту;
- серия `low_confidence` в одно время → проблема с LLM или данными;
- серия `failed` → системная ошибка tool или инфраструктуры;
- `manual` инциденты — что сотрудник вручную пометил как проблему.

## Выход

Для группы ошибок:

- общий паттерн и пример event_id;
- вероятная причина;
- рекомендация исправления (что именно и где).
