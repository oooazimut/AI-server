# Групповой анализ инцидентов

Используй этот скил, когда нужно разобрать несколько похожих ошибок вместе.

Порядок работы:

1. Начни с `diagnost_error_report(since_hours=N)` для общей картины.
2. Сгруппируй инциденты по похожей причине (`by_reason` в отчёте).
3. Для каждой группы проверь `recent_incidents` — есть ли общий паттерн в `request`.
4. При необходимости уточни конкретные события через `diagnost_search_events`.
5. Для каждой группы предложи отдельный патч — не смешивай разные причины в один вывод.
6. Для самой частой группы предложи regression test.
