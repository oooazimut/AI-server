# Fallback-процедура

Используй этот скил, если ни один диагностический скил не подошёл к задаче.

Порядок работы:

1. Определи, что именно нужно диагностировать.
2. Попробуй `diagnost_search_events` или `diagnost_list_incidents` — есть ли данные в хранилище.
3. Если данных нет — сообщи пользователю и попроси уточнить период или ключевые слова запроса.
4. Если данные есть, дай краткий предварительный разбор и укажи недостающие детали.
5. Не предлагай патч без основания в данных.
