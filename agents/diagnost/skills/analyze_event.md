# Анализ события диалога

Используй этот скил, когда нужно разобрать конкретный диалог оркестратора.

Порядок работы:

1. Найди событие через `diagnost_search_events` по ключевым словам запроса.
2. Восстанови цепочку: запрос → handoff_to (специалисты) → actions (tools) → response.
3. Отметь `status` и `confidence` — признаки автоматического инцидента.
4. Найди первый этап, где результат стал неправильным.
5. Предложи исправление: instructions.md, skill, routing, tool-параметры, тест или код.

Не раскрывай пользователю внутренние данные системы полностью. Для человека формируй краткое объяснение причины сбоя.
