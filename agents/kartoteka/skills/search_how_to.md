# Поиск файлов в Картотеке

## Когда применять

Используй этот скил, когда пользователь просит найти файл, документ, инструкцию, оборудование, настройку или любую другую позицию из локального каталога организации.

## Порядок работы

1. Сформулируй поисковый запрос кратко и без лишних слов — объект, оборудование, тема, название файла.
2. Сначала вызови `kartoteka_context`, если в tool_results ещё нет статистики каталога. Если каталог пуст — сообщи без поиска.
3. Вызови `kartoteka_search` с поисковым запросом. По умолчанию `limit=5`; увеличивай до 10–20, если нужно шире.
4. Если первый поиск дал мало результатов, попробуй другие ключевые слова из запроса пользователя.
5. Не добавляй факты, которых нет в tool_results. Не придумывай файлы.

## Как формировать ответ

- Если найден один уверенный результат — дай краткое описание и путь к файлу.
- Если найдено несколько — дай список из 3–5 лучших с именем и путём.
- Если ничего не найдено — честно сообщи и предложи уточнить запрос (другое слово, тег, расширение).

## Показ пути к файлу

Пути из каталога показывай пользователю — они публичны. Формат: коротко, без лишнего JSON.

Пример:
```
Источник: /documents/objects/Объект-12/Инструкция_эксплуатации.pdf
```

Агент не открывает и не отправляет содержимое файлов — только путь из индекса.

## Что не делать

- Не искать вне tool (в ФС, в Bitrix-диске и т.д.).
- Не раскрывать внутренний JSON, tool calls, схему маршрутизации.
- Не превращать догадки в факты.
- Этот агент не работает с Bitrix-диском — для этого есть ИИ-Битрикс.
