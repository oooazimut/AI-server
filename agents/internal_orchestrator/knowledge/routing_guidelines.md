# Правила маршрутизации запросов

## Какого специалиста выбирать

| Тема запроса | Специалист |
|---|---|
| Задачи Bitrix24: создание, комментарии, смена статуса, дедлайны | `call_bitrix24` |
| Диск Bitrix24: поиск файлов, загрузка документов | `call_bitrix24` |
| Уведомления и сообщения в Bitrix24-чатах | `call_bitrix24` |
| Технические регламенты, ПТО-документы, сравнение смет | `call_pto` |
| Служебные автомобили: заявки, утренние отчёты, учёт | `call_logistics` |

## Граничные случаи

- **«Покажи документ к задаче»** — сначала `call_bitrix24` для поиска задачи, потом `call_pto` для документа (два вызова в одном шаге)
- **«Что нового в портале?»** — `call_bitrix24`, он умеет искать по задачам и диску
- **«Сравни смету из Bitrix с нашим регламентом»** — два специалиста параллельно: `call_bitrix24` + `call_pto`
- **«Как дела?» / «Спасибо»** — отвечать самому, `tool_calls=[{"name":"none"}]`
- **Неоднозначный запрос** — уточнить у пользователя, не гадать

## Когда вызывать нескольких специалистов

Вызывай параллельно, если запрос явно касается нескольких доменов. Не вызывай одного и того же специалиста дважды в одном цикле — это признак ошибки.

## Когда отвечать самому

- Общие вопросы о системе, боте, возможностях
- Приветствия, благодарности
- Запрос уже полностью обработан предыдущими tool_results
