# Задачи: поиск и фильтры

Читать этот раздел перед поиском задач, заявок, открытых/закрытых задач, задач по сотруднику, проекту, статусу или периоду.

## Оглавление

- Термины
- Статусы задач
- Базовый поиск
- Карточка задачи
- Пользователи в фильтрах
- Временные фильтры
- Проекты

## Термины

- "Заявки" в бытовой речи сотрудников часто означает задачи.
- "Открытые задачи" или "открытые заявки" означает незавершённые задачи.
- "В работе" означает все открытые задачи, включая "ждёт выполнения" и "выполняется".
- "Выполненные", "закрытые", "завершённые" означает завершённые задачи.
- "На контроле" означает задачи со статусом "ждёт контроля".
- Если пользователь не сказал "мои", не ограничивать поиск только текущим пользователем.

## Статусы задач

- `1` - новая
- `2` - ждёт выполнения
- `3` - выполняется
- `4` - ждёт контроля
- `5` - завершена
- `6` - отложена
- `7` - отклонена

## Базовый поиск

Для живых данных Bitrix использовать `bitrix_api` с `tasks.task.list`.

Частые поля:
- `ID`
- `TITLE`
- `STATUS`
- `RESPONSIBLE_ID`
- `CREATED_BY`
- `DEADLINE`
- `GROUP_ID`
- `DESCRIPTION`
- `CLOSED_DATE`
- `CREATED_DATE`
- `CHANGED_DATE`

Пример списка открытых задач:

```json
{
  "action": "call",
  "method": "tasks.task.list",
  "params": {
    "filter": {"!STATUS": 5},
    "select": ["ID", "TITLE", "STATUS", "RESPONSIBLE_ID", "CREATED_BY", "DEADLINE"],
    "order": {"DEADLINE": "ASC"}
  }
}
```

## Карточка задачи

Для полной карточки задачи использовать `tasks.task.get`.

```json
{
  "action": "call",
  "method": "tasks.task.get",
  "params": {
    "taskId": 7797,
    "select": ["ID", "TITLE", "DESCRIPTION", "STATUS", "RESPONSIBLE_ID", "CREATED_BY", "DEADLINE"]
  }
}
```

## Пользователи в фильтрах

Если пользователь назвал исполнителя, сначала найти сотрудника через `user.search`, затем использовать его ID в фильтре задач.

Если пользователь говорит "я", "мои", "на мне", использовать `current_user.id`.

## Временные фильтры

Если пользователь говорит "за неделю", "за месяц", "за апрель", "за 14 мая", перевести это в конкретный диапазон дат и использовать соответствующие поля задач, например `CREATED_DATE`, `CLOSED_DATE`, `DEADLINE` в зависимости от смысла запроса.

Если смысл неясен, уточнить: искать по дате создания, сроку или закрытию.

## Проекты

Если задача ищется в проекте, нужно получить `GROUP_ID`. Для поиска или изменения самого проекта читать topic `projects_crm`.
