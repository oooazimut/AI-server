# Проекты и CRM

Читать этот раздел перед действиями с проектами, рабочими группами, лидами, сделками и воронками.

## Проекты / рабочие группы

В Bitrix проекты часто идут через social network / workgroups.

`sonet_group.get` - читать проекты/рабочие группы.

```json
{
  "action": "call",
  "method": "sonet_group.get",
  "params": {
    "FILTER": {"%NAME": "Транзит"}
  }
}
```

Изменение проектов (`sonet_group.update`, `sonet_group.create`, удаление) требует подтверждения.

При создании проекта по умолчанию делать проект открытым:
- `PROJECT`: `Y`;
- `VISIBLE`: `Y`;
- `OPENED`: `Y`;
- `CLOSED`: `N`.

Если руководитель проекта явно не указан, backend подставляет `OWNER_ID=current_user.id`, чтобы проект создавался от имени пользователя, который пишет помощнику, а не от имени владельца webhook-а.

Если пользователь явно просит закрытый/приватный проект, использовать указанное пользователем значение вместо дефолта.

Если инструмент для конкретного действия с проектом отсутствует или метод запрещён политикой, говорить прямо: действие пока не подключено или запрещено.

## CRM

`crm.lead.list`, `crm.lead.get` - чтение лидов.

`crm.deal.list`, `crm.deal.get` - чтение сделок.

`crm.status.list` - статусы, стадии, справочники CRM.

Создание и изменение CRM-сущностей (`crm.lead.add`, `crm.deal.update` и т.п.) требует подтверждения и доступно только разрешённым пользователям.
