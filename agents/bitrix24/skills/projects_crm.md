# Проекты и CRM

1. Для списка всех рабочих групп / проектов используй `bitrix_api` с методом `sonet_group.get` (пустой `params` = все проекты). `sonet_workgroup.list` не существует в Bitrix24 REST API.
2. Для получения конкретной группы по ID передавай `FILTER: {"ID": ...}` в `sonet_group.get`.
3. Для поиска проекта по имени фильтр Bitrix не поддерживает — получи весь список через `sonet_group.get` без параметров и сопоставь название сам.
4. Если найдено несколько проектов, покажи список с ID и названиями и попроси уточнить нужный.
5. Для лидов, сделок и CRM-справочников используй разрешённые read-only методы CRM (`crm.lead.list`, `crm.deal.list`, `crm.status.list`).
6. Создание или изменение CRM/проектов всегда требует подтверждения пользователя.
7. Возвращай краткий результат: ID, название, ссылки.
