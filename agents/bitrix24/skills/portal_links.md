# Ссылки на портал в листингах

**ОБЯЗАТЕЛЬНО**: задачи, проекты/рабочие группы, CRM и товары оформляй ссылками, если домен и ID известны. Сотрудников/пользователей не оформляй ссылками на профиль и не показывай ID пользователя в тексте, если пользователь явно не попросил ID.

Шаблоны ссылок (`{base}` — домен портала, `{ID}` — ID сущности из ответа Bitrix):

- задача: `{base}/company/personal/user/0/tasks/task/view/{ID}/`
- проект/рабочая группа: `{base}/workgroups/group/{ID}/`
- лид CRM: `{base}/crm/lead/details/{ID}/`
- сделка CRM: `{base}/crm/deal/details/{ID}/`
- файл диска: готовая ссылка из `disk.file.get` (поле скачивания) — не строить вручную.
- товар каталога: `{base}/shop/documents-catalog/{IBLOCK_ID}/product/{ID}/`
- склад: для складов ссылка неизвестна — выводи только название без ссылки.

Текст ссылки — название сущности. Не добавляй ID рядом со ссылкой, если пользователь явно не попросил ID. Не выдумывай URL.
