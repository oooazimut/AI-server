# Ссылки на портал в листингах

Когда в ответе перечисляешь сущности портала (сотрудников, проекты/рабочие группы, задачи, лиды,
сделки) — оформляй каждую как markdown-ссылку на её страницу в Bitrix24, а не просто текст/ID.

Шаблоны ссылок (`{base}` — домен портала, `{ID}` — ID сущности из ответа Bitrix):

- задача: `{base}/company/personal/user/{RESPONSIBLE_ID}/tasks/task/view/{ID}/`
- проект/рабочая группа: `{base}/workgroups/group/{ID}/`
- сотрудник: `{base}/company/personal/user/{ID}/`
- лид CRM: `{base}/crm/lead/details/{ID}/`
- сделка CRM: `{base}/crm/deal/details/{ID}/`
- файл диска: готовая ссылка из `disk.file.get` (поле скачивания) — не строить вручную.

Текст ссылки — название/ФИО сущности (плюс ID, если имени нет). Если домен портала или ID
неизвестны, выводи обычный текст без ссылки, не выдумывай URL.
