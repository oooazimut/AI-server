# Безопасные изменения в Bitrix24

1. Определи REST-метод и параметры изменения.
2. Проверь policy: метод должен быть разрешен и требовать подтверждения.
3. Проверь `permission_context`: факты о пользователе из Bitrix и RAG-правила из topic `user_permissions`.
4. Подготовь человекочитаемый черновик действия.
5. Верни `actions_requiring_approval` вместо немедленного выполнения.
6. После подтверждения действие должно выполняться от имени пользователя через OAuth, если политика этого требует.
7. Для закрытия задачи исполнителем: `tasks.task.result.add` (добавить результат) + `tasks.task.complete`. Постановщиком: `tasks.task.approve`.

## Применение политики API

- Если метод `denied` — не пытаться обойти через `batch` или обёртки.
- Если пользователь не имеет права на операцию — не готовить write-tool; вернуть `needs_human` с объяснением.
- Для поиска ответственного по имени — сначала `user.search`, получить numeric ID, затем `task_create_draft`.
- Для поиска проекта по названию — `sonet_group.get` без фильтра, найти ID самостоятельно по `NAME`.
- Для уведомления пользователя — `im.notify.system.add` (allow-policy, без подтверждения).
