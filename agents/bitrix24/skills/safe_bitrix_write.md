# Безопасные изменения в Bitrix24

1. Определи REST-метод и параметры изменения.
2. Проверь policy: метод должен быть разрешен и требовать подтверждения.
3. Убедись, что пользователь попадает в write allowlist или в ограниченное правило создания задач.
4. Подготовь человекочитаемый черновик действия.
5. Верни `actions_requiring_approval` вместо немедленного выполнения.
6. После подтверждения действие должно выполняться от имени пользователя через OAuth, если политика этого требует.
7. Для закрытия задачи из чата используй `task_closure`: модель сама выделяет `task_id` или `task_query` и `result_text`, а backend сохраняет pending `ai_server.task_closure`.
