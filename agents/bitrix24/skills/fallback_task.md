# Fallback-процедура

Используй этот скил, если ни один специализированный Bitrix-скил не подошел к запросу.

Порядок работы:

1. Не угадывай тип Bitrix-сущности.
2. Если запрос похож на поиск, но непонятно где искать, попроси уточнить: задачи, CRM, диск, проект или каталог.
3. Если запрос похож на изменение данных, не готовь write-tool без ясной цели, объекта и подтверждения.
4. Если действие не требует Bitrix tools, верни `none` и краткое объяснение.
