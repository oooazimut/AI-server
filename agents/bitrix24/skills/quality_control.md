# Контроль качества и незавершённые предложения

## Алгоритм при source=quality_control_webhook

Когда `source="quality_control_webhook"`, в контексте уже есть `task_detail` и `task_results` — не запрашивать задачу через API повторно.

1. **Нет результата** → `tasks.task.disapprove` + комментарий (`tasks.task.commentitem.add`) с просьбой добавить результат. Конец.
2. **Есть результат** → `tasks.task.approve` + комментарий с оценкой.
   - Если выполнено **полностью** — всё.
   - Если выполнено **частично** — дополнительно: вызвать `save_incomplete_proposal` с `task_id`, `task_title`, `missing_parts` (что не сделано) и `responsible_dialog_id`. Это сохранит запись и запланирует утреннее предложение менеджеру.

## Незавершённые предложения

Если в контексте есть `pending_responsible_question` — пользователь (исполнитель) ответил на вопрос о причинах частичного выполнения. Принять ответ, сохранить пояснение.

Если в контексте есть `pending_manager_proposals` — менеджер видит утреннее предложение. Если менеджер соглашается — вызвать `task_create_draft` для новой задачи на оставшиеся работы, затем `delete_incomplete_proposal` с соответствующим `proposal_id`.
