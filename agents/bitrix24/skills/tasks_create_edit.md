# Создание и изменение задач

1. Проверь, хватает ли данных: название, описание, исполнитель, проект, срок.
2. Для MVP исполнитель распознаётся только как `на меня` / `себе` или явный Bitrix user id формата `#123`.
3. Если данных не хватает, верни `needs_clarification`.
4. Подготовь черновик `tasks.task.add` с реальными `params.fields`.
5. Не выполняй write-действие без подтверждения policy layer и пользователя.
6. По умолчанию задачи должны иметь срок; если срок не указан, backend ставит дефолт MVP.
7. Проект/группа в MVP распознаётся только как явный `проект #456`; название проекта требует отдельного resolver-а.
