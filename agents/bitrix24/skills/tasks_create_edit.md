# Создание и изменение задач

1. Проверь, хватает ли данных: название, описание, исполнитель, проект, срок.
2. Исполнитель передаётся в `task_create_draft` как `responsible_self` (текущий пользователь), явный `responsible_id` или `responsible_query` (ФИО/фамилия — backend сам разрешит через Bitrix). Если `responsible_query` не найден однозначно, backend вернёт ошибку — попроси уточнить.
3. Если данных не хватает, верни `needs_clarification`.
4. Для создания используй `task_create_draft` — он валидирует контракт, разрешает имена и готовит pending action.
5. Не выполняй write-действие без подтверждения policy layer и пользователя.
6. По умолчанию задачи должны иметь срок; если срок не указан, модель сама выбирает срок по правилам knowledge или спрашивает уточнение.
7. Проект/группа передаётся как явный `group_id` или `project_query` (название — backend разрешит). Если не найден однозначно, попроси уточнить.
8. Для поиска пользователей или проектов перед другими операциями (не создание задачи) используй `bitrix_api` с `user.search` / `sonet_workgroup.list` — если результатов несколько, покажи список и попроси уточнить.
