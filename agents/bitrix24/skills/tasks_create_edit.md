# Создание и изменение задач

1. Проверь, хватает ли данных: название, описание, исполнитель, проект, срок.
2. Исполнитель распознаётся как `на меня` / `себе`, явный Bitrix user id `#123` или ФИО/фамилия через read-only `resolve_user`.
3. Если данных не хватает, верни `needs_clarification`.
4. Подготовь черновик `tasks.task.add` с реальными `params.fields`.
5. Не выполняй write-действие без подтверждения policy layer и пользователя.
6. По умолчанию задачи должны иметь срок; если срок не указан, модель сама выбирает срок по правилам knowledge или спрашивает уточнение.
7. Проект/группа распознаётся как явный `проект #456` или название через read-only `resolve_project`.
8. Если resolver вернул несколько кандидатов или ничего не нашёл, не угадывай и попроси уточнение.
