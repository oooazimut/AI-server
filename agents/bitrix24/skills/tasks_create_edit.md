# Создание и изменение задач

1. Проверь, хватает ли данных: название, описание, исполнитель, проект, срок.
2. Если данных не хватает, верни `needs_clarification`.
3. Подготовь черновик `tasks.task.add` или другого разрешенного write-метода.
4. Не выполняй write-действие без подтверждения policy layer и пользователя.
5. По умолчанию задачи должны иметь срок и контроль результата, если политика не говорит обратное.
