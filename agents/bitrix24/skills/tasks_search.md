# Поиск задач и заявок

1. Уточни фильтр: исполнитель, постановщик, проект, срок, статус или текст запроса.
2. Если пользователь просит "мои задачи" без уточнения роли, используй `bitrix_my_tasks`.
   Он включает задачи, где текущий пользователь исполнитель, постановщик, соисполнитель или другой участник Bitrix.
3. Если данных достаточно, используй `portal_search` или `bitrix_api` с read-only методом.
4. Для остальных живых данных Bitrix используй `tasks.task.list` или `tasks.task.get`.
5. Если исполнитель назван по имени — сначала `bitrix_api → user.search`, получить ID, затем использовать в фильтре.
6. Если результатов много, верни краткую выборку и предложи уточнить фильтр.
7. Не закрывай, не меняй и не удаляй задачи в этом skill.
