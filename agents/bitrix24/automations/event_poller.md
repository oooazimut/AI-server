# Polling входящих сообщений

Источник переноса: `BitrixAIAgent/app/agent/event_poller.py`.

## Роль

Channel adapter. Получает события бота через polling/fetch режим, когда webhook
режим недоступен или временно выключен.

## Входы

- Bitrix bot events API.

## Выходы

- Нормализованные входящие сообщения в Оркестратор.
- Ответы Оркестратора обратно в Bitrix-чат.

## State

- Runtime offset последнего события.

## Правило переноса

Это альтернативный входной канал. Он не должен содержать Bitrix-экспертизу:
после парсинга сообщения поток идёт в Оркестратор.
