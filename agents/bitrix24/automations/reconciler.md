# Сверка потерянных событий

Источник переноса: `BitrixAIAgent/app/agent/reconciler.py`.

## Роль

Scheduled worker. Компенсирует случаи, когда webhook не пришёл, пришёл не в том
порядке или был пропущен во время простоя сервера.

## Входы

- Список недавно изменённых задач из Bitrix REST.
- Дельта диска через search indexer.

## Выходы

- Синтетические `ONTASKUPDATE` события в очередь.
- Запуск дельта-индексации диска.

## State

Отдельного постоянного state минимум; использует очередь webhook-событий и state
индексатора.

## Правило переноса

Reconciler не принимает бизнес-решений. Он только восстанавливает технический
событийный поток и обновляет индекс.
