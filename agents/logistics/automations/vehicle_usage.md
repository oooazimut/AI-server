# Утренний учёт служебных машин

Источник переноса: `BitrixAIAgent/app/agent/vehicle_usage.py`; целевой runtime в AI-server — детерминированная автоматизация в `backend.ai_server.agent_worker`.

LLM и самостоятельное толкование свободного текста в этом контуре отсутствуют.

## Роль

Детерминированная бизнес-автоматизация. Scheduler утром инициирует сбор
ежедневного отчёта, а заранее заданный workflow отправляет сообщения,
контролирует сроки и сохраняет структурированные данные.

## Входы

- Расписание запросов и эскалаций.
- Структурированные команды оркестратора.
- Технические события Bitrix и расписания.

## Выходы

- Напоминания ответственному сотруднику.
- Уточняющие вопросы по неполным данным.
- Структурированные записи по машинам и сотрудникам.
- Эскалации администраторам.

## Runtime

Backend-worker отвечает за техническое расписание, рабочие дни, dedupe,
заранее заданные уведомления и хранение state. Если пользовательский запрос
требует выбора действия, этот выбор делает Pro-оркестратор и передаёт точную
структурированную команду одному из `vehicle_usage_*` tools. Автоматизация
самостоятельно не толкует пользовательский текст и не меняет выбранное действие.
Отправка сообщений людям остаётся в канальном runtime.

## State

- PostgreSQL schema `logistics`.
- Основные таблицы: `vehicle_usage_requests`, `employee_daily_statuses`,
  `vehicle_daily_assignments`, `vehicle_daily_drivers`, `vehicles`,
  `employees`, `vehicle_usage_revisions`.

## Правило переноса

Bitrix здесь является каналом и техническим API. Смысл пользовательского
запроса принадлежит Pro-оркестратору, а детерминированные правила учёта —
этому workflow.
