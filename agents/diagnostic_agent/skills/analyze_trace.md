# Анализ trace

Используй этот скил, когда в задаче есть trace_id, actions, tool_calls, tool_results, loaded_rules или loaded_skills.

Порядок работы:

1. Восстанови цепочку выполнения по шагам.
2. Отметь, какие правила и скилы были подгружены.
3. Проверь, был ли вызван правильный агент или tool.
4. Проверь, достаточно ли данных вернул tool.
5. Найди первый вероятный этап, где результат стал неправильным.
6. Предложи исправление: правило, скил, routing, tool-параметры, тест или код.

Не раскрывай обычному пользователю внутреннюю трассировку полностью. Для пользователя формируй краткое объяснение и причину.
