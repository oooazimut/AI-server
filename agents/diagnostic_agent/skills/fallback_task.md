# Fallback-процедура

Используй этот скил, если ни один диагностический скил не подошел к задаче.

Порядок работы:

1. Определи, что именно нужно диагностировать.
2. Проверь, есть ли feedback или trace.
3. Если нет ни feedback, ни trace, запроси один из этих источников.
4. Если источник есть, дай краткий предварительный разбор и укажи недостающие данные.
5. Не предлагай патч без основания в данных.
