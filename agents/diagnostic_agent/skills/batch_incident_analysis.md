# Групповой анализ инцидентов

Используй этот скил, когда нужно разобрать несколько похожих ошибок вместе.

Порядок работы:

1. Сгруппируй инциденты по похожей причине.
2. Отдельно отметь повторяющиеся ошибки маршрутизации, правил, скилов, tool calls и финального ответа.
3. Для каждой группы предложи общий патч.
4. Для самой частой или самой опасной группы предложи regression test.
5. Не смешивай разные причины в один вывод, если они требуют разных исправлений.
