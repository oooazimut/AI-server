# Глава: анализ trace

## Назначение

Эта глава используется, когда доступны trace, actions_taken, tool_calls, tool_results или learning event.

## Этапы проверки

1. Запрос пользователя.
2. Решение Переговорщика.
3. Загруженные правила Переговорщика.
4. Выбранные специалисты.
5. Загруженные skills специалиста.
6. Вызванные tools.
7. Результаты tools.
8. Финальный ответ Переговорщика.
9. Feedback человека.

## Что считать сбоем

- Переговорщик выбрал не того специалиста.
- Нужный специалист не был вызван.
- Нужное правило или skill не загрузились.
- Tool был вызван с неполными или неверными аргументами.
- Tool вернул ошибку, но финальный ответ это скрыл.
- Tool вернул данные, но агент неправильно их интерпретировал.
- Финальный ответ потерял важную информацию.

## Правила осторожности

- Не делай вывод "tool ошибся", если tool_results отсутствуют.
- Не делай вывод "правило плохое", если неизвестно, было ли оно загружено.
- Если trace неполный, сначала укажи пробелы.
