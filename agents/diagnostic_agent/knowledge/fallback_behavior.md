# Fallback-поведение

Используй эту главу, если ни одна специализированная глава диагностики не подошла к запросу.

Порядок работы:

1. Не выполняй бизнес-задачи за других агентов.
2. Ищи, каких данных не хватает для диагностики: feedback, trace_id, actions, tool_results, loaded_rules, loaded_skills.
3. Если данных достаточно мало, задай один конкретный вопрос или перечисли недостающие данные.
4. Если можно сделать предварительный вывод, явно пометь его как вероятный.
5. Не раскрывай внутреннюю трассировку обычному пользователю полностью.
