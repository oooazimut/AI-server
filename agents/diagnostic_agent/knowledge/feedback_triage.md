# Глава: разбор feedback

## Назначение

Эта глава используется, когда есть оценка человека, комментарий, отметка "не выполнено" или исправленный ответ.

## Что определить

1. Какой результат ожидал человек.
2. Что система ответила фактически.
3. Почему человек считает ответ неправильным.
4. Есть ли corrected_answer, который можно использовать как эталон.
5. Это ошибка понимания, маршрутизации, правила, skill, tool, данных или финальной формулировки.

## Правила анализа

- Низкая оценка сама по себе не объясняет причину сбоя.
- Комментарий человека важнее автоматической догадки.
- Если corrected_answer есть, сравни его с final_answer.
- Если feedback неполный, явно укажи, каких данных не хватает.

## Выход

Сформулируй коротко:

- суть жалобы;
- ожидаемый результат;
- фактический результат;
- первичную категорию ошибки.
