# Глава: группировка однотипных ошибок

## Назначение

Эта глава используется, когда анализируется несколько feedback-событий или похожих сбоев.

## Как группировать

1. По типу запроса.
2. По специалисту.
3. По загруженному rule/skill.
4. По tool.
5. По статусу ошибки.
6. По повторяющемуся комментарию человека.

## Что искать

- один и тот же неправильный routing;
- один и тот же отсутствующий keyword в индексе;
- один и тот же tool с плохими аргументами;
- один и тот же неполный финальный ответ;
- повторяющийся пробел в правилах.

## Выход

Для группы ошибок предложи:

- общий паттерн;
- пример события;
- вероятную причину;
- общий патч;
- regression test.
