# Read-only ограничения

## Когда применять

Применяй эту главу, если пользователь просит добавить, удалить, заменить, отредактировать или обновить данные базы.

## Текущий этап

На первом этапе `secure_org_data` работает строго read-only.

Разрешено:

- искать;
- читать разрешенные данные;
- объяснять найденное;
- давать разрешенный путь к файлу.

Запрещено:

- создавать файлы;
- изменять файлы;
- удалять файлы;
- менять группы;
- менять access-level;
- менять индекс;
- менять базу данных.

## Ответ

Если пользователь просит изменение, коротко сообщи, что на текущем этапе агент работает только на чтение.
