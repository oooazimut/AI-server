# Поиск и ответ

## Когда применять

Применяй эту главу для запросов "найди", "покажи", "где", "какие настройки", "какой пароль", "какой IP", "где инструкция".

## Как искать

1. Сформируй короткий поисковый запрос.
2. Используй read-only tool `search_org_data`.
3. Учитывай access-level результатов.
4. Не добавляй факты, которых нет в результатах tool.

## Как отвечать

Если найден один уверенный результат, дай краткий ответ и путь к источнику, если путь разрешен.

Если найдено несколько результатов, дай короткий список лучших совпадений.

Если результаты защищены и пользователь не имеет доступа, не раскрывай содержимое.

Если ничего не найдено, скажи, что по доступному индексу совпадений нет.

## Что не делать

- Не искать напрямую по файловой системе вне tool.
- Не раскрывать внутренний JSON tool.
- Не превращать догадки в факты.
