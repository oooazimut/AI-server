# Bitrix24 workers and automations

## Граница ответственности

Смысл входящего запроса определяет только внутренний оркестратор. Он выбирает
инструмент, точные параметры, порядок действий и итоговый формат ответа.
`Bitrix24Specialist` принимает только версионированную структурированную команду,
проверяет её контракт, права текущего пользователя, OAuth и выполняет указанный
Bitrix API-вызов.

Bitrix workers не являются агентами и не принимают бизнес-решений. Им разрешены
только технические функции:

- приём и дедупликация webhook-событий;
- обновление поискового индекса;
- восстановление пропущенных событий индексирования;
- постановка закрытой задачи в оркестраторский контур проверки;
- технические retry, idempotency и аудит.

Удалённые legacy-контуры `bitrix_task_supervisor`,
`bitrix_task_quality_control` и свободнотекстовые «предложения» не являются
частью runtime и не должны возвращаться без отдельного решения пользователя.

## Активные контуры

| Контур | Назначение |
| --- | --- |
| `bitrix_chat_adapter` | Приём сообщений Bitrix и передача их оркестратору |
| `bitrix_webhook_event_queue` | Очередь, dedupe и retry webhook-событий |
| `bitrix_portal_search_indexer` | Периодическое обновление PostgreSQL-индекса |
| `bitrix_search_webhook_indexer` | Быстрое обновление индекса по disk/file webhook |
| `bitrix_reconciler` | Техническое восстановление пропущенных событий |
| `bitrix_task_close_direct_dispatcher` | Передача события закрытия задачи оркестратору |

## Поиск и PostgreSQL

PostgreSQL хранит технический поисковый снимок. Workers обновляют его по
расписанию и событиям, но не выбирают сущности за пользователя. Оркестратор
выбирает точные идентификаторы из собственного каталога возможностей и сущностей.
Перед выдачей защищённых результатов Bitrix-исполнитель повторно проверяет доступ
текущего пользователя.

## Запрещённые пути

- свободный текст из worker или специалиста непосредственно в Bitrix API;
- LLM, supervisor или quality-control внутри Bitrix-исполнителя;
- самостоятельная смена инструмента либо параметров исполнителем;
- скрытые бизнес-дефолты и сопоставление имён внутри Bitrix tools;
- фоновое выполнение бизнес-действий в обход оркестратора.

Каноническое правило закреплено в корневом
`ORCHESTRATOR_AUTHORITY.md` и защищается архитектурными тестами.
