# Архитектура

## Слои

```text
Каналы общения
  ↓
Channel Adapters
  ↓
Оркестраторы / операторы
  ↓
Agent Registry
  ↓
Специалисты
  ↓
Tool Gateway
  ↓
Битрикс24 / 1С / Excel / Git / сети / файлы
```

## Основные компоненты

### Channel Adapters

Адаптеры принимают сообщения из Битрикс24, MAX, веб-чата или другого канала и приводят их к единому формату.

### Оркестраторы

Оркестратор не должен знать бухгалтерию, сети или ПТО в деталях. Он знает реестр специалистов, их возможности и политики вызова.

### Agent Registry

Реестр хранит манифесты агентов: роль, возможности, инструменты, доступы и действия, требующие подтверждения.

### Tool Gateway

Все внешние действия проходят через ограниченные инструменты. Инструменты должны возвращать структурированные результаты.

### Model Gateway

Шлюз моделей выбирает локальную или облачную LLM по типу данных и задаче. Чувствительные данные не отправляются наружу без явного разрешения политики.

### Audit Log

Все обращения, вызовы инструментов, решения и подтверждения пишутся в журнал.
