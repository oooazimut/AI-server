# Безопасность и политики

## Базовые правила

- Минимальные права доступа для каждого агента и инструмента.
- Разделение внутренних и клиентских контуров.
- Клиентский оператор не видит внутренние комментарии и чувствительные данные.
- Опасные действия требуют подтверждения человека.
- Все действия пишутся в аудит.

## Автоматически разрешенные действия L1

- read-only диагностика;
- проверка статуса устройства;
- перезапуск камеры;
- синхронизация времени;
- сбор логов;
- создание черновика заявки.

## Действия с подтверждением

- изменение IP, шлюза, DNS;
- изменение паролей;
- изменение NAT, VPN, firewall;
- обновление прошивки;
- сброс настроек;
- удаление записей;
- проведение документов в 1С;
- отправка документов клиентам или контрагентам.

## Лимиты автоустранения

- максимум 2-3 автоматических действия на обращение;
- общий лимит времени 3-5 минут;
- не повторять одно и то же действие по одному устройству слишком часто;
- при неопределенности создавать заявку.

## Администрирование знаний

- Редактирование skills/RAG через чат разрешается только отдельному
  админскому LLM-специалисту.
- Модель может предлагать изменения, но запись на диск идёт только через
  ограниченный patch-tool и после подтверждения человека.
- Старый формат `project_shell` как произвольная консоль из чата не переносится.
- Запрещён доступ к `.env`, OAuth SQLite, webhook-токенам, runtime-аудиту и
  другим секретам.
